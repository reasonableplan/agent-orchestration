---
name: impact
description: Change impact analysis — find all files affected by a modification before making it
---

<Purpose>
코드 변경 전에 영향 범위를 분석한다. "이 파일을 바꾸면 어디가 깨지는지" 미리 파악한다.
</Purpose>

<Use_When>
- 변경 영향도 확인: "이거 바꾸면 어디 영향?", "/impact"
- 인터페이스/타입 변경 전
- 함수 시그니처 변경 전
- 모듈 삭제 전
</Use_When>

<Steps>
1. **대상 파악**: 변경하려는 파일/함수/타입 식별
2. **의존성 추적**:
   - **Import 분석**: 이 모듈을 import하는 모든 파일 (Grep)
   - **호출처 분석**: 변경할 함수/클래스를 사용하는 모든 곳
   - **타입 의존성**: 변경할 타입을 참조하는 모든 곳
   - **테스트 의존성**: 관련 테스트 파일과 mock
3. **영향도 분류**: 직접 영향 / 간접 영향 / 테스트 영향
4. **수정 체크리스트 생성**

### 출력 형식
```
## Impact Analysis: [변경 대상]

### 직접 영향 (반드시 수정 필요)
1. `file.py:42` — [영향받는 이유]

### 간접 영향 (확인 필요)
1. `service.py:15` — [영향 가능성]

### 테스트 영향
1. `test_file.py` — [수정/추가 필요한 테스트]

### 수정 체크리스트
- [ ] 타입 정의 수정
- [ ] 구현 수정
- [ ] mock 업데이트
- [ ] 테스트 수정/추가
- [ ] 호출처 업데이트

### 리스크
- [놓치면 런타임에 깨질 수 있는 부분]
```
</Steps>
